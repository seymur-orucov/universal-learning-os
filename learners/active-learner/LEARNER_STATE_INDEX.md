## What to Update After Each Session

- Update `EVIDENCE_LOG.md` when learner output was captured, summarized, reviewed, or assessed.
- Update `REVIEW_QUEUE.md` when weak learning skills or repeated mistakes are identified.
- Update `NEXT_ACTIONS.md` with the next recommended command and reason.
- Update `SESSION_HISTORY.md` with the session summary.
- Update `STATE.md` only when evidence or explicit user instruction justifies a state change.

## What to Paste Into a New Chat

Use `learners/active-learner/SESSION_HANDOFF_CURRENT.md` as the compact handoff file for ChatGPT continuation.

If a previous chat was deleted, paste the latest handoff plus relevant evidence summaries. Do not pretend deleted chat contents are known.
