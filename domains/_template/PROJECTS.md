# Projects

## Purpose

Define realistic projects for a domain pack.

## Project Structure

Each project SHOULD include:

- Project id: stable project identifier.
- Title: human-readable project name.
- Purpose: why the project exists.
- Prerequisites: required concepts or learning skills.
- Milestones: staged project checkpoints.
- Artifacts: expected learner-produced outputs.
- Review criteria: criteria for reviewing artifacts.
- Evidence value: what the project can demonstrate when reviewed.
- Related learning skills: learning skills practiced or assessed by the project.

## Normative Requirements

- Artifact existence alone MUST NOT imply mastery.
- Project artifacts MAY become evidence when reviewed.
- Projects SHOULD reference learning skills from `domains/_template/SKILL_GRAPH.md`.
- Projects MUST NOT duplicate learner artifacts into reusable domain pack content.
