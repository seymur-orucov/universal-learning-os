# Practice Rules

## Purpose

Define how this domain pack maps core practice types to subject-specific practice.

## Practice Type Mapping

- Recall practice: describe suitable recall tasks for the domain.
- Guided practice: describe scaffolded task patterns.
- Independent practice: describe unsupported application tasks.
- Mixed practice: describe how multiple learning skills are combined.
- Error-based practice: describe how common mistakes trigger targeted practice.
- Project-based practice: describe artifact-producing practice.
- Interview-style practice: describe question-and-answer practice.

## Normative Requirements

- Practice rules SHOULD reference `core/learning-engine/PRACTICE_MODEL.md`.
- Practice tasks SHOULD produce evidence when progress or mastery may be updated.
- Practice rules MUST NOT contain learner-specific progress.
- Practice rules SHOULD reference learning skills from `domains/_template/SKILL_GRAPH.md`.
