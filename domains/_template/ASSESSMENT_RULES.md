# Assessment Rules

## Purpose

Define how this domain pack maps core assessment dimensions to domain-specific rubrics.

## Assessment Dimension Mapping

- Correctness: define what correct work means in this domain.
- Independence: define how support, hints, or scaffolding affect evaluation.
- Explanation quality: define expected reasoning or communication.
- Transfer ability: define realistic or novel application contexts.
- Consistency: define repeated-performance expectations.
- Speed: define timing expectations only if domain-appropriate.

## Normative Requirements

- Assessment rules SHOULD reference `core/learning-engine/ASSESSMENT_MODEL.md`.
- Assessment rules MAY map `core/mastery-model/MASTERY_LEVELS.md` to domain-specific rubrics.
- Assessment rules MUST NOT weaken evidence requirements from `core/mastery-model/EVIDENCE_REQUIREMENTS.md`.
- Speed MUST NOT be used unless domain-appropriate.
- Assessment rules MUST NOT contain learner-specific mastery records.
