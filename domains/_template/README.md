# Domain Pack Template

This directory is a reference structure for future domain packs. It is not a real learning domain and MUST NOT be treated as reusable subject content.

## Files

- `DOMAIN.md`: domain identity, audience, goals, outcomes, tracks, localization policy, prerequisites, and boundaries.
- `SYLLABUS.md`: staged learning sequence with modules, lessons, objectives, practice, assessments, projects, and review points.
- `SKILL_GRAPH.md`: learning skill definitions, prerequisites, relationships, mastery targets, evidence, and glossary links.
- `PRACTICE_RULES.md`: mapping from core practice types to domain-specific practice.
- `ASSESSMENT_RULES.md`: mapping from core assessment dimensions to domain-specific rubrics.
- `PROJECTS.md`: project definitions, milestones, artifacts, review criteria, and evidence value.
- `GLOSSARY.md`: terminology guidance supporting instruction language and terminology language preferences.

## Relationships

- Domain pack rules are defined in `specification/DOMAIN_PACK_SPEC.md`.
- Localization rules are defined in `specification/LOCALIZATION_SPEC.md`.
- Evidence requirements are defined in `core/mastery-model/EVIDENCE_REQUIREMENTS.md`.
