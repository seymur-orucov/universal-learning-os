# Glossary

## Purpose

Define terminology guidance for a domain pack.

## Glossary Entry Structure

Each glossary entry SHOULD include:

- Term: important domain-specific term.
- Terminology language: preferred language for the term.
- Short explanation: concise meaning.
- Optional translation or clarification in instruction language.
- Examples: usage examples.
- Usage notes: preservation, ambiguity, or style guidance.
- Related learning skills: learning skills connected to the term.

## Normative Requirements

- Glossary guidance MUST follow `specification/LOCALIZATION_SPEC.md`.
- Important technical terms SHOULD NOT be replaced with unnatural translations.
- Glossary entries SHOULD support separate instruction language and terminology language preferences.
- A project preference MAY use Azerbaijani instruction with English technical terms, but this MUST NOT be hardcoded as the global framework language.
