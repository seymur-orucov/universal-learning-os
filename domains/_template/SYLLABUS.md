# Syllabus

## Purpose

Define the expected learning sequence for a domain pack.

## Structure

Each syllabus SHOULD define:

- Stages: major progression bands or phases.
- Modules: coherent groups of related lessons or learning skills.
- Lessons: instructional units mapped to learning objectives.
- Learning objectives: what the learner should practice or demonstrate.
- Prerequisites: required prior concepts or learning skills.
- Practice expectations: expected practice types and frequency.
- Assessment checkpoints: points where evidence should be collected.
- Projects or milestones: realistic artifacts or integrated tasks.
- Review points: planned review opportunities for weak or stale learning skills.

## Normative Requirements

- Lesson completion MUST NOT imply mastery.
- Syllabus entries SHOULD reference learning skills from `domains/_template/SKILL_GRAPH.md`.
- Assessment checkpoints SHOULD align with `domains/_template/ASSESSMENT_RULES.md`.
- Review points SHOULD align with `core/learning-engine/REVIEW_MODEL.md`.
- The syllabus MUST NOT contain learner-specific progress.
