# JavaScript Domain Pack

This is the JavaScript pilot domain pack for the Stage 11 domain pack factory model.

It contains reusable subject-specific content for learning JavaScript from beginner foundations through advanced/professional runtime reasoning, async behavior, browser basics, engineering practices, and interview-style explanation. It is separate from learner state and MUST NOT contain learner-specific progress or mastery records.

## Localization

This domain SHOULD use Azerbaijani learner-facing explanations by default while preserving JavaScript technical terms in English where natural.

## Files

- `DOMAIN.md`: standard domain identity, scope, audience, outcomes, and boundaries.
- `SYLLABUS.md`: staged `javascript.main` learning path.
- `SKILL_GRAPH.md`: JavaScript learning skills, dependencies, and review priorities.
- `PRACTICE_RULES.md`: evidence-producing practice rules.
- `ASSESSMENT_RULES.md`: checkpoint and evidence rules.
- `PROJECTS.md`: project-based learning opportunities.
- `GLOSSARY.md`: JavaScript terminology guidance for Azerbaijani learner-facing instruction.

## Supporting Files

- `DOMAIN_OVERVIEW.md`: earlier domain summary retained for traceability.
- `MENTOR_BEHAVIOR.md`: learner-facing teaching behavior and clean output expectations.
- `PRACTICE_MODEL.md`: compact practice model retained for Project Pack derivation.
- `ASSESSMENT_MODEL.md`: compact assessment model retained for Project Pack derivation.
- `MASTERY_EXPECTATIONS.md`: domain-specific mastery expectations mapped to framework evidence rules.
- `PROJECT_PACK_SELECTION.md`: generated Project Pack selection and file budget guidance.

## Relationships

- Domain pack rules are defined in `specification/DOMAIN_PACK_SPEC.md`.
- Domain creation process is defined in `docs/domain-pack-factory.md`.
- Project Pack export rules are defined in `exports/project-pack-spec/`.
