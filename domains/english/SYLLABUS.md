# English Syllabus

## Syllabus Overview

This syllabus defines the `english.main` learning path for English for Software Engineers. It maps stages, modules, and concise lesson outlines to learning skills from `domains/english/SKILL_GRAPH.md`.

This file defines the learning path only. It does not contain full lesson bodies, dialogues, listening transcripts, shadowing scripts, exercise banks, learner state, or learner progress.

Lesson completion does not imply mastery. Speaking participation does not imply speaking mastery. Generated dialogues, scripts, or shadowing prompts do not imply mastery. Mastery requires captured, summarized, or reviewed learner output as evidence. Learner progress belongs in learner state, not this syllabus.

## Syllabus Conventions

- Lesson outlines are intentionally concise and MUST NOT be treated as full lessons.
- Target learning skills use ids from `domains/english/SKILL_GRAPH.md`.
- Practice expectations SHOULD follow `domains/english/PRACTICE_RULES.md`.
- Assessment checkpoints SHOULD follow `domains/english/ASSESSMENT_RULES.md`.
- Review points SHOULD align with `core/learning-engine/REVIEW_MODEL.md`.
- Next actions SHOULD align with `core/learning-engine/NEXT_ACTION_MODEL.md`.
- English target examples, phrases, collocations, interview answers, and learner outputs SHOULD remain in English where appropriate.

## Track: `english.main`

- Purpose: B1-to-C1 speaking-first professional English for software engineers.
- Scope: speaking, grammar accuracy, vocabulary activation, listening, shadowing, pronunciation, workplace communication, technical explanation, writing, interview English, review, and maintenance.
- Non-goal: this track is not a full IELTS/TOEFL course, a dialogue bank, or a grammar-only syllabus.

- Stage goal: Build confidence producing short English output about familiar work and personal-professional contexts.
- Included learning skills: `english.speaking-confidence`, `english.self-introduction`, `english.daily-routine-speaking`, `english.simple-opinion-expression`, `english.asking-for-clarification`, `english.basic-self-correction`, `english.reducing-hesitation`.

## Review and Assessment Strategy

- Review SHOULD target weak learning skills, stale learning skills, failed assessments, repeated errors, and upcoming workplace or interview needs.
- Assessment MUST be based on evidence, not lesson exposure or participation.
- Speaking, listening, shadowing, writing, interview, and portfolio work SHOULD produce captured, summarized, or reviewed learner output before mastery recommendations.
- Portfolio milestones MAY support higher mastery only when reviewed and connected to learning skills.
- State updates SHOULD be proposed rather than silently applied in agent-mediated workflows.
