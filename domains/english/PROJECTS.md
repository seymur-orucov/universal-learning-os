# English Portfolio and Project Guidance

## Projects Overview

This file defines reusable portfolio and project guidance for the `english` domain pack. It supports future speaking practice, interview preparation, workplace communication, writing review, pronunciation practice, listening practice, and evidence-backed assessment.

This file defines structured placeholders only. It does not create full project tasks, speaking task banks, interview question banks, dialogue banks, listening transcripts, shadowing scripts, lesson bodies, learner artifacts, or learner state.

## Project Principles

- Projects MUST be reusable domain pack content, not learner state.
- Projects MUST NOT record learner progress, learner mastery, or private learner artifacts.
- Project categories SHOULD reference learning skills from `domains/english/SKILL_GRAPH.md`.
- Project output SHOULD follow `templates/session/PROJECT_OUTPUT_TEMPLATE.md` when used in a session.
- Portfolio work SHOULD support evidence-based practice without requiring audio storage by default.
- Project guidance MUST NOT weaken evidence requirements from `core/mastery-model/EVIDENCE_REQUIREMENTS.md`.

## Portfolio Difficulty Guidance

- P1: Guided mini artifact; learner produces a short artifact with prompts or scaffolding.
- P2: Independent focused artifact; learner produces a narrow artifact with minimal hints.
- P3: Integrated workplace artifact; learner combines speaking, writing, listening, or interaction in a realistic context.
- P4: Reviewed portfolio artifact; learner produces an artifact suitable for structured review, revision, and reflection.
- P5: Defended interview or speaking artifact; learner explains, responds to follow-up, or defends choices under review.

Project or portfolio difficulty is not learner mastery. Mastery MUST be recorded only in learner state when evidence or explicit user instruction supports it.

- Project id: `english.project.speaking-portfolio`
- Title: Speaking Portfolio
- Purpose: Collect reviewed speaking outputs that show growth in confidence, fluency, accuracy, and clarity.
- Suggested stage range: Stage 1 through Stage 12, with formal portfolio use in Stage 11.
- Related learning skills:
  - `english.speaking-confidence`
  - `english.self-introduction`
  - `english.work-status-updates`
  - `english.fluency-accuracy-pressure`

## Portfolio Evidence Rules

- Artifact existence alone does not imply mastery.
- Reviewed artifacts MAY support evidence.
- Speaking recordings, transcripts, summaries, or notes MAY support evidence only when reviewed.
- Project work MAY support M5 or M6 only when transfer, explanation, interaction, consistency, and review quality are sufficient.
- Portfolio progress belongs in learner state, not this domain pack.
- Audio storage is not required by default and privacy must be considered.
- Portfolio sessions MAY propose learner state updates but MUST NOT silently apply them.

## Review Criteria

Portfolio review SHOULD consider:

- Correctness: grammar, vocabulary, meaning, and task fit.
- Independence: amount of prompting, scaffolding, or translation support needed.
- Explanation quality: structure, clarity, relevance, detail, and audience fit.
- Transfer ability: whether learner output works in new workplace, technical, interview, listening, or writing contexts.
- Consistency: whether repeated outputs show stable performance and reduced errors.
- Evidence quality: whether the artifact is captured, summarized, or reviewed clearly enough to support state proposals.
- Privacy: whether artifact storage avoids unnecessary personal, workplace, audio, or transcript data.

## Project-to-Syllabus Mapping

- Stage 1 speaking confidence: Speaking Portfolio SHOULD capture baseline and confidence-building artifacts.
- Stage 3 status updates: Workplace Communication Portfolio SHOULD include status updates, blocker explanations, and help requests.
- Stage 5 listening, shadowing, and pronunciation: Listening Comprehension Portfolio and Shadowing and Pronunciation Log SHOULD capture reviewed summaries and pronunciation notes.
- Stage 6 technical explanation: Technical Explanation Portfolio SHOULD collect concept, architecture, bug, and tradeoff explanations.
- Stage 7 meetings and feedback: Workplace Communication Portfolio SHOULD include meeting, clarification, feedback, and decision-summary artifacts.
- Stage 8 writing: Professional Writing Portfolio SHOULD collect emails, chat messages, PR comments, summaries, and revisions.
- Stage 9 interview: Software Engineering Interview Portfolio SHOULD collect behavioral, technical, follow-up, and uncertainty-handling responses.
- Stage 10 B2/C1 fluency: Speaking Portfolio and Technical Explanation Portfolio SHOULD include longer explanations and defended responses.
- Stage 11 portfolio: all portfolio categories MAY be used as structured milestones.
- Stage 12 review and maintenance: Error Correction and Self-Correction Log SHOULD guide review and maintenance.

## Privacy and Evidence Notes

- Portfolio guidance MUST NOT require storing raw audio, transcripts, or private workplace content by default.
- Summaries, anonymized notes, or reviewer observations MAY be sufficient when they preserve evidence value.
- Real workplace artifacts SHOULD be sanitized before use.
- Private learner data belongs in learner state or learner-owned storage, not in this reusable domain pack.
- Missing evidence MUST be reported honestly.

## Localization Notes

- Feedback and explanations MAY be delivered in Azerbaijani when learner or project preferences require it.
- English target examples, phrases, collocations, interview answers, and learner outputs SHOULD remain in English where appropriate.
- Azerbaijani instruction with English target language is a project preference, not a global framework default.
- Localization behavior SHOULD follow `specification/LOCALIZATION_SPEC.md`.
