# SQL/PostgreSQL Practice Rules Draft

This file defines concise draft practice guidance. It does not contain full exercises.

## Practice Type Mapping

- Recall practice: syntax, definitions, and key distinctions such as `primary key` vs `foreign key`, `WHERE` vs `HAVING`, and `JOIN` types.
- Guided practice: scaffolded query writing with partial prompts, table hints, and step-by-step reasoning.
- Independent practice: learner writes queries without hints against a known schema or dataset.
- Mixed practice: combine filtering, joins, grouping, subqueries, CTEs, and ordering in one task.
- Error-based practice: repair wrong queries, incorrect joins, null-handling mistakes, aggregation errors, or misconceptions.
- Project-based practice: schema, query, and reporting tasks using realistic requirements.
- Interview-style practice: explain query behavior, schema choices, tradeoffs, and performance reasoning.

## Dataset Guidance

Northwind is a useful realistic dataset for SQL/PostgreSQL practice where appropriate. This draft does not create full Northwind exercises yet.
