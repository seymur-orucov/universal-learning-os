# SQL/PostgreSQL Assessment Rules Draft

This file maps core assessment dimensions to SQL/PostgreSQL draft rubric guidance.

## Assessment Dimension Mapping

- Correctness: query returns the intended result and handles relevant edge cases.
- Independence: learner writes or explains the query without excessive hints.
- Explanation quality: learner can explain query logic, table relationships, and database behavior.
- Transfer ability: learner can apply concepts to new tables, schemas, or requirements.
- Consistency: learner performs reliably across related query or schema tasks.
- Speed: relevant only for interview or fluency contexts, not primary mastery.

## Normative Notes

- Assessment rules SHOULD follow `core/learning-engine/ASSESSMENT_MODEL.md`.
- Assessment MUST be based on evidence.
- These rules MUST NOT weaken evidence requirements from `core/mastery-model/EVIDENCE_REQUIREMENTS.md`.
- Speed MUST NOT be used as a primary mastery requirement unless the user goal or context makes fluency relevant.
