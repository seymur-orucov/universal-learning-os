# Terminology

This document defines shared vocabulary. Normative behavior belongs in the relevant `specification/` file.

## Terms

- Framework: the domain-independent learning system contracts.
- Domain pack: reusable subject-specific learning material and metadata.
- Learner state: learner-specific progress, evidence, preferences, and history.
- Skill: reusable agent capability that can support learning workflows.
- Command: user-invoked workflow entry point.
- Evidence: observable learner activity used to justify state changes.
- Mastery: a learner-state claim that a skill or concept has been demonstrated to the required standard.
- Canonical source: the single document that defines a concept's normative contract.

## RFC Keywords

The words `MUST`, `MUST NOT`, `SHOULD`, `SHOULD NOT`, and `MAY` are used to indicate requirement levels in specification documents.
