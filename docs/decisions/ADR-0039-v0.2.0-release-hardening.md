# ADR-0039: v0.2.0 Release Hardening

## Status

Accepted for Stage 17.0.

## Context

The repository now has four supported domains, standard and compact generated Project Packs, CLI inspection, CLI validation, compact generation, and standard generation. After implementing generators, the main release risk is no longer missing pack artifacts; it is inconsistent documentation, unclear release checks, and accidental drift between generated packs, launch kits, and canonical sources.

## Decision

Stage 17.0 hardens the repository for a clean `v0.2.0` release. It updates release notes, checklists, user-facing documentation, generated-pack guidance, and manual smoke tests without adding a new learning domain or a major feature.

## Stable for v0.2.0

- Supported domains: `sql-postgresql`, `english`, `javascript`, and `typescript`.
- Standard Project Packs: exactly 25 files per supported domain.
- Compact Project Packs: exactly 5 files per supported domain.
- CLI commands for listing domains, inspecting packs, validating packs, and generating standard/compact packs.
- Learner-Facing Mentor Mode and metadata visibility guardrails in generated packs.
- Launch kits for practical ChatGPT Project setup.

## Experimental or Future Work

- Machine-readable schemas and deep semantic validation.
- Runtime application/dashboard support.
- Automated learner state update workflow.
- Source drift checks and manifest checksums.
- English audio/pronunciation privacy workflow.
- Additional domains beyond the current four.

## Release Readiness Model

Generated packs are release artifacts derived from canonical repository files. The CLI generators can recreate standard and compact packs, and `node tools/ulos-cli/src/index.js validate` is the official quality gate before release.

Validation does not prove complete semantic equivalence, but it verifies the required pack directories, file counts, required files, Learner-Facing Mentor Mode marker, metadata guardrails, standard manifest basics, compact structure, and launch kit presence.

## No New Domain

No new domain is added in Stage 17.0 because release hardening should stabilize existing contracts before expanding the supported surface area. New domains should follow the post-v0.2.0 workflow in `docs/domain-pack-factory.md`.
