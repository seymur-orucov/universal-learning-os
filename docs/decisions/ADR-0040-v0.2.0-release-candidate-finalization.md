# ADR-0040: v0.2.0 Release Candidate Finalization

## Status

Accepted for Stage 17.1.

## Context

Stage 17.0 hardened repository documentation for `v0.2.0` after standard and compact generators were implemented. A separate finalization stage is needed to produce explicit release-candidate status, tagging instructions, and a final manual acceptance test without expanding scope.

## Decision

Stage 17.1 finalizes the repository as a `v0.2.0` release candidate. Finalization means the release documentation, checklist, validation report, and manual acceptance test clearly state what must pass before tagging.

## Ready for v0.2.0

- Four supported domains: `sql-postgresql`, `english`, `javascript`, and `typescript`.
- Standard profile generated packs: exactly 25 files each.
- Compact profile generated packs: exactly 5 files each.
- CLI listing, inspection, validation, compact dry-run generation, and standard dry-run generation.
- Launch kits for standard and compact ChatGPT Project setup.
- Learner-Facing Mentor Mode and metadata visibility guardrails in generated packs.

## Deferred Work

- Runtime learner-state implementation.
- App/dashboard work.
- Additional domains.
- Package distribution.
- Stronger automated tests, source drift checks, checksums, and schema validation.

## No New Feature or Domain

No new feature or domain is added in this stage because release-candidate finalization should reduce risk, not expand the supported surface. New domains and runtime work belong after `v0.2.0`.
