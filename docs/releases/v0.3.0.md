# v0.3.0 Release Notes

## Release Summary

Universal Learning OS v0.3.0 adds optional learner runtime CLI helpers, automated CLI tests, and the local Studio MVP control panel.

ChatGPT Projects remain the daily learning runtime. Codex and the repository remain the framework maintenance environment. Studio is a local control panel, not a full LMS. No new domain is added in v0.3.0, and generated pack contracts remain unchanged.

## Major Additions

- Optional learner runtime strategy documentation, handoff exports, and snapshot templates.
- CLI helpers for learner handoff scaffolds, learner snapshot scaffolds, and learner artifact validation.
- Automated CLI regression tests using Node's built-in test runner.
- Local Universal Learning OS Studio MVP under `apps/studio/`.
- Studio hardening docs, smoke scripts, health checks, bridge rejection clarity, and boundary-focused UI copy.

## CLI Updates

The CLI now supports the release baseline commands:

```sh
node tools/ulos-cli/src/index.js list-domains
node tools/ulos-cli/src/index.js validate
node tools/ulos-cli/src/index.js inspect-pack --domain typescript --profile standard
node tools/ulos-cli/src/index.js inspect-pack --domain typescript --profile compact
node tools/ulos-cli/src/index.js generate --domain typescript --profile standard --dry-run
node tools/ulos-cli/src/index.js generate --domain typescript --profile compact --dry-run
node tools/ulos-cli/src/index.js create-handoff --domain typescript --out learners/active-learner/handoffs/example.md
node tools/ulos-cli/src/index.js create-snapshot --domain typescript --type weekly --out learners/active-learner/snapshots/example.md
node tools/ulos-cli/src/index.js validate-learner
```

`validate` remains the generated pack quality gate. The learner runtime helpers remain optional and user-requested.

## Learner Runtime Updates

- Daily learning remains chat-local in ChatGPT Projects.
- Repository learner state remains optional and snapshot-based.
- Handoff and snapshot artifacts are optional and user-requested.
- `validate-learner` checks optional learner artifacts for forbidden internal-default markers.
- No mastery is inferred automatically from chat history.
- No full ChatGPT transcript import, YAML learner-state patch generation, mandatory daily learner-state workflow, or heavy runtime learner-state system is added.

## Automated Testing Updates

CLI automated tests are available from `tools/ulos-cli/`:

```sh
npm test
```

The Stage 20.0 suite covers generated pack validation, domain listing, TypeScript standard and compact pack inspection, generation dry-runs, invalid inputs, path safety, overwrite protection, learner handoff/snapshot helpers, and learner artifact validation.

## Studio MVP Updates

Studio is a local React/Vite UI with a local Node CLI bridge. It is intended as a control panel for existing CLI-backed workflows.

- The bridge uses allowlisted actions only.
- The bridge invokes CLI commands with argument arrays.
- The bridge uses `shell: false`.
- Studio does not expose arbitrary shell execution.
- Studio does not add authentication, cloud storage, database storage, ChatGPT API integration, transcript import, generated pack editing, automatic mastery inference, or a full LMS runtime.

## Generated Pack Contract

- Standard packs remain exactly 25 files per supported domain.
- Compact packs remain exactly 5 files per supported domain.
- Compact packs retain the same exact file list:
  - `PROJECT_INSTRUCTIONS.md`
  - `STARTUP_PROMPT.md`
  - `DOMAIN_CORE.md`
  - `COMMANDS_CORE.md`
  - `MENTOR_SKILLS_CORE.md`
- Generated packs remain learner-neutral.
- Learner-Facing Mentor Mode remains unchanged.

## Supported Domains

- `sql-postgresql`
- `english`
- `javascript`
- `typescript`

No new domain is added in v0.3.0.

## Validation Commands

Run from the repository root:

```sh
node tools/ulos-cli/src/index.js validate
node tools/ulos-cli/src/index.js validate-learner
node tools/ulos-cli/src/index.js list-domains
node tools/ulos-cli/src/index.js inspect-pack --domain typescript --profile standard
node tools/ulos-cli/src/index.js inspect-pack --domain typescript --profile compact
node tools/ulos-cli/src/index.js generate --domain typescript --profile standard --dry-run
node tools/ulos-cli/src/index.js generate --domain typescript --profile compact --dry-run
```

Run automated CLI tests:

```sh
cd tools/ulos-cli
npm test
cd ../..
```

Run Studio checks:

```sh
cd apps/studio
npm install
npm run build
npm run smoke:bridge
npm run smoke:health
cd ../..
```

Finish with:

```sh
git diff --check
git status --short
```

## Upgrade Notes From v0.2.0

- Learner runtime strategy is documented as optional, snapshot-based, and evidence-first.
- Handoff and snapshot templates are available for user-requested learner continuity.
- CLI learner helpers add `create-handoff`, `create-snapshot`, and `validate-learner`.
- Automated CLI tests are available through `npm test` in `tools/ulos-cli/`.
- Studio MVP provides a local control panel for existing CLI workflows.
- Studio hardening adds local usage docs, smoke checks, health checks, and clearer bridge error output.
- Generated pack contracts are unchanged from v0.2.0.

## Known Limitations

- Manual acceptance tests remain Markdown checklists.
- Studio automated UI tests are future work.
- Studio is local-only and must be run by a trusted local user.
- Learner artifacts are optional scaffolds, not a complete learner-state database.
- Source drift detection and machine-readable release manifests remain future work.

## Non-Goals

- Add a new domain.
- Change standard or compact generated pack contracts.
- Replace ChatGPT Projects as the daily learning runtime.
- Require repository updates after every lesson.
- Implement a heavy runtime learner-state system.
- Import or parse full ChatGPT transcripts.
- Infer mastery automatically from chat history.
- Generate YAML learner-state patches.
- Store sensitive personal data.
- Add authentication, cloud storage, database storage, or ChatGPT API integration.
- Build a full LMS.
