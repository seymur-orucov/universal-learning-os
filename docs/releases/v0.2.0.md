# v0.2.0 Release Notes

## Summary

`v0.2.0` prepares Universal Learning OS for practical generated Project Pack use across four domains with CLI validation and generation support.

## Supported Domains

- `sql-postgresql`
- `english`
- `javascript`
- `typescript`

## Pack Profiles

- `standard`: exactly 25 files per domain for normal ChatGPT Project usage.
- `compact`: exactly 5 files per domain for Free Project or low-file-budget usage.

Generated packs live under `exports/generated/`. Launch kits live under `exports/generated/project-launch-kits/`.

## ChatGPT Project Usage

Use the relevant launch kit to create a Project, upload the generated pack files, and paste the ready startup prompt. Standard packs provide more inspectable context. Compact packs merge the essentials into five files.

## CLI Commands

Run from the repository root:

```sh
node tools/ulos-cli/src/index.js list-domains
node tools/ulos-cli/src/index.js validate
node tools/ulos-cli/src/index.js inspect-pack --domain typescript --profile standard
node tools/ulos-cli/src/index.js inspect-pack --domain typescript --profile compact
node tools/ulos-cli/src/index.js generate --domain typescript --profile standard
node tools/ulos-cli/src/index.js generate --domain typescript --profile compact
node tools/ulos-cli/src/index.js generate --domain typescript --profile standard --dry-run
node tools/ulos-cli/src/index.js generate --domain typescript --profile standard --out-dir tmp/typescript-standard-test
```

## Validation Quality Gate

Before release or after generation, run:

```sh
node tools/ulos-cli/src/index.js validate
```

Expected result: all eight generated packs pass.

## Upgrade Notes From v0.1.0-rc1

- JavaScript and TypeScript are now supported domains.
- Compact/free packs exist for all supported domains.
- Standard and compact packs can be generated by the CLI.
- CLI validation checks file counts, required files, metadata guardrails, manifests, compact structure, and launch kits.
- Learner-Facing Mentor Mode is consistently documented in generated packs.

## Known Limitations

- Manual acceptance tests are still Markdown guides, not executable tests.
- Machine-readable schemas and deep source-drift detection are future work.
- Learner state updates remain manual and evidence-first.
- Generated packs are reusable and learner-neutral; personal learner state must remain separate.
- App/dashboard/runtime state strategy remains future work.
