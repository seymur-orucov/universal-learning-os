# GoF and React Design Patterns Real Learning Pilot

## Status

Planned; do not mark complete without a real learner session and reviewed artifacts.

## Goal

Evaluate whether the domain creates pattern-selection and simplification judgment rather than catalog recall.

## Pilot Path

1. Start in compact profile with no learner state; complete Module 1 guided check and one TypeScript/Vitest baseline.
2. Study one pattern from each GoF category: Factory Method, Adapter and State.
3. Require an OO vs functional/no-pattern comparison and a pattern-removal task.
4. Study compound components, state colocation, adapter hooks and error boundary; include at least two RTL rendered-behavior tests.
5. Complete one refactoring lab and one code-review mode interview.
6. Move to standard profile using a grounded handoff; verify no progress is invented.
7. Complete one case study milestone sequence and an unseen transfer task.
8. Inspect Studio dynamic domain listing and standard/compact pack health.

## Evidence to Collect

- Learner problem/forces notes before pattern naming.
- TypeScript implementation and meaningful Vitest tests.
- Purposeful JavaScript runtime comparison or explicit reason it adds no value.
- React rendered-behavior test where applicable.
- Rejected alternative and no-pattern threshold.
- Refactor diff with safety-net test and pattern removal.
- Diagram explanation, including simplification.
- Interview response and post-attempt feedback.
- Transfer solution in a different application context.

## Evaluation Questions

- Does the learner explain why a pattern fits, not merely identify it?
- Can the learner choose a simpler functional/data design?
- Can the learner find misuse and safely remove abstraction?
- Do tests protect behavior rather than participant/class names?
- Are React analogies technically bounded?
- Do hints preserve productive struggle?
- Does either profile leak internal metadata or infer mastery?

## Stop Conditions

Pause and record a gap if a lesson reveals a solution before attempt, requires untaught prerequisites, presents a false GoF–React equivalence, uses brittle tests, treats pattern count as quality, or mutates learner progress without evidence.
