# GoF and React Design Patterns Acceptance Checklist

## Discovery and Contracts

- [ ] `list-domains` shows `gof-react-patterns` exactly once.
- [ ] Canonical directory has exactly nine learner-neutral Markdown files.
- [ ] Standard inspect reports exactly 25 files and five `GOF_REACT_PATTERNS_*` outputs.
- [ ] Compact inspect reports exactly five files: `COMMANDS_CORE.md`, `DOMAIN_CORE.md`, `MENTOR_SKILLS_CORE.md`, `PROJECT_INSTRUCTIONS.md`, `STARTUP_PROMPT.md`.
- [ ] All eight launch-kit files exist.

## Teaching Depth and Technical Accuracy

- [ ] First lesson teaches problem, naive design, forces and simplest candidate before a pattern name.
- [ ] Sample creational, structural, behavioral and React lessons follow the 28-part contract across turns.
- [ ] TypeScript implementation is complete; JavaScript comparison explains a real runtime difference.
- [ ] Functional/data-driven and no-pattern alternatives are viable, not strawmen.
- [ ] Vitest tests assert public behavior and failure paths.
- [ ] React Testing Library appears only where a component is rendered and uses semantic user behavior.
- [ ] GoF–React links are labeled conceptual and include differences.
- [ ] Mermaid blocks render and educational simplifications are labeled.

## Judgment and Evidence

- [ ] Learner can reject a suggested pattern with context-specific evidence.
- [ ] Learner can remove an existing pattern under safety-net tests.
- [ ] Misuse, overengineering, testability and removal triggers appear in feedback.
- [ ] Assessment requires implementation, tests, explanation, trade-offs and transfer.
- [ ] Pattern-name recall, lesson exposure and copied solution do not produce mastery.
- [ ] Complete solutions remain gated until an attempt or explicit request.

## Commands and Profiles

- [ ] `START_TRACK`, `START_LESSON`, `CONTINUE_LESSON`, `PRACTICE`, `REVIEW`, `ASSESS`, `START_PROJECT`, `INTERVIEW` and `SHOW_PROGRESS` behave as documented.
- [ ] Standard and compact first lessons both start with foundations without state.
- [ ] Normal teaching hides internal skill IDs, YAML, evidence blocks and pack internals.
- [ ] Missing learner state/progress is reported honestly.
- [ ] Studio lists the domain through CLI discovery; no domain-specific bridge/UI behavior is required.
