# Learner Handoff Example

This is a concise example for explicit handoff or continuation use. It does not include sensitive personal data and does not require a repository learner-state update.

## Domain

SQL/PostgreSQL

## Current Lesson or Topic

Filtering rows with `WHERE`, comparison operators, and combining conditions with `AND` / `OR`.

## Completed Topics

- Basic `SELECT` structure.
- Selecting specific columns.
- Simple `WHERE` filters.
- Basic comparison operators: `=`, `<>`, `>`, `<`, `>=`, `<=`.

## Current Strengths

- Understands the basic order of `SELECT ... FROM ... WHERE`.
- Can write simple filters for one condition.
- Uses SQL keywords consistently.

## Current Weak Areas

- Needs more practice grouping mixed `AND` / `OR` conditions.
- Sometimes forgets that string values need quotes.
- Needs review of how `NULL` differs from an empty string or zero.

## Last Completed Task

The learner wrote a query to select customers from a table where `country = 'Azerbaijan'` and reviewed why the string literal needs quotes.

## Recommended Next Action

Practice 3 short filtering tasks using `AND`, `OR`, and parentheses, then review mistakes before introducing `IS NULL`.

## Mentor Behavior Reminders

- Explain in Azerbaijani while keeping SQL/PostgreSQL terms in English where natural.
- Keep normal learner-facing output clean.
- Do not show evidence/state/proposed update metadata unless explicitly requested.
- Do not mark mastery from lesson exposure alone.
- Ask the learner to write queries, not only read generated examples.
