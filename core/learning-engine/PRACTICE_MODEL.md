# Practice Model

## Purpose

Define domain-independent practice types.

## Scope

This model defines practice categories. Domain packs define concrete prompts, tasks, constraints, and expected answers.

## Practice Types

- Recall practice: asks the learner to retrieve facts, definitions, steps, or distinctions.
- Guided practice: supports the learner through a task with hints, scaffolding, or partial structure.
- Independent practice: asks the learner to complete a task with minimal support.
- Mixed practice: combines multiple learning skills or concepts.
- Error-based practice: targets observed mistakes or misconceptions.
- Project-based practice: asks the learner to produce a realistic artifact.
- Interview-style practice: simulates verbal, written, or interactive questioning.

## Normative Requirements

- Practice SHOULD produce evidence usable for feedback or assessment.
- Practice difficulty SHOULD be selected from learner state, domain pack prerequisites, and recent evidence.
- Error-based practice SHOULD reference the mistake or evidence that triggered it.
- Project-based practice MAY span multiple sessions.
- Practice tasks MUST NOT duplicate learner state into reusable domain content.

## Relationships

- Assessment dimensions are defined in `ASSESSMENT_MODEL.md`.
- Review triggers are defined in `REVIEW_MODEL.md`.
- Domain-specific practice rules belong in domain packs.
