# TypeScript Domain Core

Canonical sources: `domains/typescript/DOMAIN.md`, `SYLLABUS.md`, `SKILL_GRAPH.md`, `PRACTICE_RULES.md`, `ASSESSMENT_RULES.md`, `PROJECTS.md`, `GLOSSARY.md`, and `QUALITY_REVIEW.md`.

## Goal

Help the learner write, debug, refactor, review, and explain practical TypeScript for frontend/backend engineering, React projects, API typing, large codebases, and interviews.

## Track

- Foundations: TypeScript vs JavaScript, static typing, annotations, primitives, arrays, tuples, objects, optional/readonly properties, functions, `void`, `never`, `unknown`, `any`, literal types, inference.
- Core type system: `type`, `interface`, `type` vs `interface`, unions, intersections, narrowing, guards, discriminated unions, exhaustiveness, `never`.
- Generics: generic functions/interfaces/aliases, constraints, defaults, `keyof`, indexed access, utility helpers, common mistakes.
- Modern/practical TypeScript: `as const`, `satisfies`, type-position `typeof`, template literal types, enum vs union literals, module typing, declaration files, ambient types.
- Utility and advanced types: `Partial`, `Required`, `Readonly`, `Pick`, `Omit`, `Record`, `Exclude`, `Extract`, `NonNullable`, `ReturnType`, `Parameters`, `Awaited`, mapped types, conditional types, distributive conditionals, `infer`, recursive/branded types.
- Engineering: `tsconfig`, `strict`, strictness flags, path aliases, module resolution, Vite/React/Node, type checking vs transpilation, ESLint TypeScript basics.
- React/API readiness: props, children, events, forms, refs, hooks, generic components, API data, TanStack Query conceptually, DTOs, request/response typing, validation boundaries, typed errors/results, API versioning.
- Interview readiness: explain-output, type-error fixing, `any` refactors, domain modeling, generic helper design, bad TypeScript review, architecture scenarios, large-codebase typing strategy.

## Evidence

Valid evidence includes reviewed typed code, type-error fixes, explanations, refactors, React/API/configuration tasks, code review comments, and interview answers. Generated examples are not evidence by themselves.
