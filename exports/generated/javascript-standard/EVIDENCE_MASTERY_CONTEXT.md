# Evidence and Mastery Context

Mastery requires evidence or explicit user instruction.

## Valid JavaScript Evidence

- Reviewed learner code.
- Reviewed output prediction with reasoning.
- Reviewed bug fix.
- Reviewed explanation of concepts such as `closure`, `this`, `prototype`, `Promise`, or `event loop`.
- Reviewed frontend scenario solution.

## Not Evidence by Itself

- Reading a lesson.
- Receiving generated examples.
- Copying code.
- Uploading files.
- Saying the topic is familiar.

## Metadata Visibility

Evidence and proposed state updates should be shown only when the learner asks for progress/state details or asks for a session summary.
