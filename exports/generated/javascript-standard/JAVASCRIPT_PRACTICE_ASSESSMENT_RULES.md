# JavaScript Practice and Assessment Rules

## Practice

- Use output prediction, bug repair, code writing, refactoring, testing, async tracing, performance reasoning, security review, and interview explanation.
- Require learner reasoning.
- Prefer small examples that reveal one mental model issue.
- Use frontend scenarios when useful.

## Assessment

Accepted evidence includes reviewed learner code, predictions, explanations, bug fixes, refactors, tests, async traces, performance reasoning, safe DOM/security reasoning, and frontend/backend scenario solutions.

Do not accept lesson exposure, copied solutions, generated examples, or project setup as mastery evidence.

## Checkpoints

Assess correctness, reasoning, independence, transfer, and repair. If a learner answers correctly but cannot explain, treat it as partial evidence.

## Clean Output

Normal practice and assessment feedback should not show evidence blocks, YAML updates, mastery tables, or implementation details unless explicitly requested.
