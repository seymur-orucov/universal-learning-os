# Evidence and Mastery Context

Canonical sources: `core/mastery-model/` and `specification/LEARNING_LIFECYCLE.md`.

## Valid TypeScript Evidence

- Reviewed typed code.
- Reviewed type-error fixes.
- Reviewed explanations of inference, narrowing, generics, utility types, or advanced types.
- Reviewed React/API/configuration tasks.
- Reviewed refactors from `any` to safer types.
- Reviewed interview answers and code review comments.

## Not Evidence by Itself

- Reading a lesson.
- Receiving generated examples.
- Copying code.
- Uploading files or setting up a Project.

Evidence and proposed state updates should be shown only when explicitly requested or when the learner asks for a session summary.
