# TypeScript Practice and Assessment Rules

Canonical sources: `domains/typescript/PRACTICE_RULES.md`, `domains/typescript/ASSESSMENT_RULES.md`, and `domains/typescript/QUALITY_REVIEW.md`.

## Practice

Use type annotation tasks, type-error fixing, type modeling, narrowing practice, generic helper design, `any` refactors, code review, React/API typing scenarios, and interview explanations.

## Assessment

Accepted evidence includes reviewed typed code, type-error fixes, explanations, refactors, React/API/configuration tasks, code review comments, and interview answers.

Do not accept lesson exposure, copied solutions, generated examples, or project setup as mastery evidence.

## Checkpoints

Assess correctness, safety, reasoning, readability, transfer, and repair. If code compiles but the learner cannot explain the type model, treat it as partial evidence.

## Clean Output

Normal practice and assessment feedback should not show evidence blocks, YAML updates, mastery tables, or implementation details unless explicitly requested.
