# Command: CONTINUE_LESSON

## Purpose

Continue an in-progress SQL/PostgreSQL lesson or session path.

## When to Use

Use when a prior lesson, practice item, assessment, or next action exists in learner state or a session report.

## Required Behavior

- Resume from learner state, session history, or the user-provided summary.
- Avoid repeating prior instruction unless review is requested or evidence shows confusion.
- Continue with practice, quick check, assessment, or next action depending on state.
- Re-check prerequisites when the continuation point is unclear.

## Output Expectations

Summarize the continuation point, deliver the next useful activity, and end with evidence summary and next action.

## Evidence and State Handling

Continuation does not imply progress by itself. State updates require learner output, reviewed answer, assessment result, project artifact, interview response, or explicit user instruction.

## Localization Expectations

Use Azerbaijani instruction and keep SQL/PostgreSQL terminology in English when natural.
