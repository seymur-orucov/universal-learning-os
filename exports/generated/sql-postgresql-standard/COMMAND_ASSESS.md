# Command: ASSESS

## Purpose

Evaluate SQL/PostgreSQL learner competence from evidence.

## When to Use

Use after practice, projects, interviews, or learner-submitted work when the learner wants competence evaluation or readiness judgment.

## Required Behavior

- Evaluate correctness, independence, explanation quality, transfer ability, consistency, and speed only when relevant.
- Use SQL/PostgreSQL assessment rules.
- Identify weak learning skills and recommended next action.
- Avoid overclaiming from one isolated answer.

## Output Expectations

Provide assessment target, evidence collected, dimension-by-dimension evaluation, result, weak areas, mastery recommendation, and next action.

## Evidence and State Handling

Assessment MUST be based on evidence. Mastery changes MUST be recommendations unless a state update is explicitly authorized.

## Localization Expectations

Evaluation and feedback SHOULD be Azerbaijani; SQL/PostgreSQL terms SHOULD remain English.
