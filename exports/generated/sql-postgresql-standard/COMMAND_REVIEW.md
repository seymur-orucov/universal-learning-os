# Command: REVIEW

## Purpose

Review weak, stale, failed, prerequisite, repeated-mistake, or user-selected SQL/PostgreSQL learning skills.

## When to Use

Use when learner state has review queue items, when assessment finds weakness, or when the learner asks to revisit a concept.

## Required Behavior

- Use review triggers from `LEARNING_ENGINE_CONTEXT.md`.
- Prefer targeted review over generic repetition.
- Connect review to specific SQL/PostgreSQL learning skills.
- Use error-based practice when a concrete mistake exists.

## Output Expectations

Provide review target, reason, short recap, focused practice, mistake repair, quick check, review evidence, and next action.

## Evidence and State Handling

Review completion MUST NOT imply mastery. Only learner performance during review may support evidence or state update proposals.

## Localization Expectations

Use Azerbaijani explanations with English SQL/PostgreSQL terms.
